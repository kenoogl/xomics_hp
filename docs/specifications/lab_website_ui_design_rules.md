# lab_website_ui_design_rules.md

## 目的

本ドキュメントは、MIT・Stanford・ETH・Berkeley・DeepMind など
世界トップ研究室サイト約20件を分析して抽出した
**研究室サイトのUI設計ルール**をまとめたものである。

研究室サイトの目的は

* 研究内容を明確に伝える
* 論文や研究成果にアクセスできる
* 学生や共同研究者を引きつける

ことである。

基本思想：

* シンプル
* 研究中心
* 高速
* 長期運用可能

---

# 基本原則

トップ研究室サイトの共通原則

```
研究情報 ＞ デザイン装飾
```

重要な優先順位

1 研究内容の理解
2 情報構造
3 ナビゲーション
4 更新の容易さ

---

# 1 ヒーローセクション

トップページでは
**研究内容が3秒以内に理解できる必要がある。**

構造

```
研究室名
↓
研究ミッション（1〜2行）
```

例

```
AI for Science Laboratory
シミュレーションとAIによる科学発見の加速
```

訪問者は

```
この研究室は何を研究しているか
```

をすぐ理解できる必要がある。

---

# 2 ナビゲーション構造

メニュー数

```
最大6項目
```

推奨構成

```
Home
Research
Publications
Members
News
Join
```

避けるもの

* 多階層メニュー
* 項目が多すぎるナビゲーション

目標

```
すべての情報に3クリック以内で到達
```

---

# 3 Researchページ中心設計

研究室サイトでは

```
Research
```

が最も重要なページである。

構造

```
Research
   ├ 研究テーマ1
   ├ 研究テーマ2
   ├ 研究テーマ3
```

各研究ページの構造

```
研究課題
↓
研究アプローチ
↓
代表論文
↓
図
```

研究説明は必ず入れる。

---

# 4 Publicationsページ

論文は

```
年ごとに整理
```

構造

```
2026
2025
2024
```

各論文の表示

```
Title
Authors
Venue
Year
PDF / DOI
```

原則

```
1行 = 1論文
```

余計な装飾は不要。

---

# 5 図による研究説明

研究テーマには必ず図を入れる。

例

```
シミュレーション結果
研究フロー図
システム構成図
```

原則

```
研究テーマページには最低1つの図
```

---

# 6 ミニマルデザイン

トップ研究室サイトの共通デザイン

```
白背景
黒文字
アクセントカラー
```

アクセントカラーは

```
大学カラー
```

を使用することが多い。

---

# 7 最新論文表示

トップページには

```
Latest Publications
```

を配置する。

表示数

```
3〜5件
```

目的

```
研究活動が継続していることを示す
```

---

# 8 メンバーページ

メンバー情報

```
写真
名前
役職
メール
```

写真サイズ

```
200×200
```

学生も写真付きが望ましい。

---

# 9 Newsセクション

ニュースは簡潔に。

例

```
論文がNeurIPSに採択
学生がBest Paper Award受賞
```

長い記事は不要。

---

# 10 研究室紹介ページ

研究室紹介は簡潔に。

推奨

```
2〜3段落
```

内容

```
研究分野
研究方針
共同研究分野
```

---

# 11 研究成果リンク

研究成果ページでは

```
論文
コード
データ
スライド
```

をリンクする。

例

```
Paper | Code | Dataset | Slides
```

---

# 12 GitHub連携

計算科学・AI研究室では

```
GitHub
```

リンクを掲載する。

例

```
論文 → GitHub
```

---

# 13 Joinページ

学生募集ページを設ける。

内容

```
研究テーマ
応募方法
連絡先
```

ページタイトル例

```
Join Us
Prospective Students
```

---

# 14 URL設計

URLは短くする。

例

```
/research
/publications
/members
/news
/join
```

深いURL構造は避ける。

---

# 15 モバイル対応

レスポンシブデザイン必須。

スマホでも

```
Research
Publications
Members
```

が見える必要がある。

---

# 16 高速サイト

トップ研究室サイトの特徴

```
JavaScriptが少ない
```

推奨

```
静的サイト
```

避ける

```
重いフロントエンド
```

---

# 17 研究知識ネットワーク

強い研究室サイトは

```
研究テーマ
↓
論文
↓
コード
```

を相互リンクする。

これは

```
Research Knowledge Graph
```

になる。

---

# 18 フッター情報

フッターには

```
研究室名
大学
所在地
メール
```

を記載する。

---

# 19 更新の可視化

サイトの信頼性は

```
最新論文
最新ニュース
```

で判断される。

更新が見える構造にする。

---

# 20 広告禁止

研究室サイトでは

```
広告
SNSウィジェット
```

は不要。

研究情報に集中する。

---

# まとめ

トップ研究室サイトの設計原則

```
研究中心
シンプル
高速
```

研究室サイトは

```
研究コミュニケーションのプラットフォーム
```

として設計する。
